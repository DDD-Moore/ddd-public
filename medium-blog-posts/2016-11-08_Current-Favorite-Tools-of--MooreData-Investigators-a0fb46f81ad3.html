Current Favorite Tools of #MooreData Investigators

By Carly Strasser

You might wonder what it is that Foundation staff do all year (other than swimming in piles of cash like Uncle Scrooge). One of our many activities includes bringing together our grantees so that they can learn from one another — and we can learn from them. We just wrapped up one such event: our 2016 Investigator Symposium. We invited the 14 DDD Investigators, and they brought one lab member each, making the total count of around 30 folks talking about data-driven science.

As part of this three day meeting, we ask each of the attendees to describe a current favorite tool or method. We (and they) get to hear about the latest interesting technology or software that’s making their lives easier. In the interest of sharing the knowledge, we’ve put this list below with links. Go forth and explore, and of course add tools to the comments that you think are missing!

  • Pyfilesystem: simplified common interface to a variety of different filesystems, such as the local filesystem, zip files, ftp servers etc.
  • Slack: messaging app for teams (especially Github & Bitbucket integrations)
  • Dask: Parallel computing library for Python
  • Coveralls & Coverage: python testing packages
  • Julia: high-level, high-performance computing language
  • Rescue Time: time management software
  • TypeScript: superset of JavaScript that compiles to plain JavaScript
  • Neo4j: native graph database that leverages data and data relationships
  • workflowy: notebook for lists
  • taxize: taxonomic toolbelt for R
  • draw.io: drawing program
  • Pydoit package: command line workflow automation. Links to examples and a tutorial available here
  • Apple pencil for iPad
  • CrashPlan: computer backup software
  • Jupyter notebooks
  • self-organizing map: type of artificial neural network
  • Trello: Project management software
  • ShareLaTeX: online LaTeX editor
  • Overleaf: collaborative writing and publishing platform
  • Paperpile: reference management
  • bokeh: interactive visualization library for Python
  • keras: high-level neural networks library, written in Python
  • Terra: low-level system programming language that is embedded in and meta-programmed by the Lua programming language.
  • Continuous integration systems for science (see this paper by Beaulieu-Jones & Greene), and micro-services
  • checkpoint: package reproducibility in R
  • Data Package specification from Open Knowledge International
